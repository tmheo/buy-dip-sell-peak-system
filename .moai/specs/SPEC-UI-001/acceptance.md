# SPEC-UI-001 인수 기준

## 시나리오 1: TopControlBar 모바일 레이아웃

**Given** 사용자가 375px~768px 너비의 모바일 기기에서 접속하고
**When** 로그인 상태에서 어떤 페이지든 방문하면
**Then** TopControlBar의 모든 버튼이 화면 내에 표시되고 overflow가 발생하지 않아야 한다

**Given** 사용자가 769px 이상의 화면에서 접속하면
**When** 어떤 페이지든 방문하면
**Then** TopControlBar가 기존 데스크톱 레이아웃 그대로 표시되어야 한다

## 시나리오 2: 인라인 폼 모바일 스택

**Given** 사용자가 모바일 기기(768px 이하)에서 추천전략 페이지에 접속하고
**When** 폼 영역을 확인하면
**Then** 제목과 폼이 수직으로 분리되고, 입력 필드가 전체 너비로 표시되어야 한다

**Given** 사용자가 모바일에서 백테스트(기본) 페이지의 폼을 사용할 때
**When** 날짜 입력 필드를 터치하면
**Then** 최소 44px 높이의 터치 영역이 확보되어 쉽게 선택할 수 있어야 한다

## 시나리오 3: 계좌 목록 모바일 카드 뷰

**Given** 로그인한 사용자가 모바일 기기로 트레이딩 계좌 목록 페이지에 접속하고
**When** 계좌가 2개 이상 등록되어 있을 때
**Then** 테이블 대신 카드 형태로 계좌 목록이 표시되어야 한다
**And** 각 카드에서 계좌 이름, 종목, 전략, 시드, 보유 상태를 확인할 수 있어야 한다
**And** "자세히" 버튼을 터치하여 계좌 상세로 이동할 수 있어야 한다

**Given** 같은 사용자가 데스크톱(769px 이상)에서 접속하면
**When** 계좌 목록 페이지를 방문하면
**Then** 기존 테이블 형태 그대로 표시되어야 한다

## 시나리오 4: 보유현황 모바일 카드 뷰

**Given** 사용자가 모바일에서 트레이딩 계좌 상세 페이지에 접속하고
**When** 보유현황 섹션을 확인하면
**Then** 9컬럼 테이블 대신 티어별 카드 형태로 보유현황이 표시되어야 한다
**And** 보유 중인 티어는 시각적으로 강조(색상/테두리)되어야 한다

## 시나리오 5: 수익현황 총계 카드 모바일

**Given** 사용자가 모바일에서 수익 현황을 확인할 때
**When** 총계 카드 영역을 확인하면
**Then** 5개 항목이 2~3열로 재배치되어 가독성이 유지되어야 한다
**And** 금액과 수익률 텍스트가 잘리지 않아야 한다

## 시나리오 6: Desktop 레이아웃 무변경 확인

**Given** 사용자가 1920px 데스크톱 화면에서 접속하면
**When** 모든 7개 페이지를 순회하면
**Then** 기존 데스크톱 레이아웃이 변경 없이 동일하게 표시되어야 한다

## 비기능 요구사항

- 새로운 JavaScript 의존성 추가 없음
- CSS 추가량이 기존 globals.css(383줄) 대비 50% 이내
- 모든 변경은 @media (max-width: 768px) 내부에 격리
